

### Purpose
This dataset was created to document the Bayesian Black-Box Optimisation (BBO) capstone project at Imperial College London. It contains sequential evaluations of eight synthetic black-box functions used to benchmark optimisation strategies.The dataset enables reflection on Bayesian optimisation strategies, surrogate modelling choices, and acquisition function behaviour across multiple benchmark functions.


### Task
The dataset supports sequential optimisation tasks where evaluations are costly and information about the objective function is limited. It is used to design and evaluate strategies that balance exploration and exploitation under query constraints.

### Creators
The dataset was created by me as part of the ICL ML capstone project requirements.We were given a handful of initial points to get an idea of the search space which was later enriched by our queries.


## 1. Composition

### Dataset Contents
The dataset consists of recorded optimisation queries for eight synthetic black-box functions. Each row represents a single function evaluation.The dataset is stored in CSV format and there are no missing/NaN values. Data points are sequentially related: each successive query depends on previous evaluations used to inform the surrogate model’s predictions.

### Data Fields
Common fields include:
- `function_id`: Identifier of the black-box function
- `round`: The optimisation iteration
- `x1, x2, …`: Input variables
- `objective_value`: Observed output

## 2. Collection Process

### Data Generation
Queries were generated by running Bayesian optimisation using a surrogate model across the eight benchmark functions.  

### Query Strategy
Initial queries were selected after building a Gaussian Process Regressor model in Scikit-Learn with random hyperparameters. Subsequent queries used model-based selection guided by acquisition functions such as EI, PI, and UCB/LCB.

### Time Frame
Data was collected over the course of the project timeline (13 weeks).

### Ethics and Consent
The dataset contains synthetic evaluations and no manual data was collected.

## 3. Preprocessing and Uses

### Preprocessing
Before storage, input vectors were normalised to a fixed range ( [0, 1]). Surrogate model inputs were checked each round for consistency and validity(NaN/Missing Values,values being out of the possible search space).

### Initial Data Preservation
Both initial and the sequentially modified dataset is preserved and uploaded here separately
